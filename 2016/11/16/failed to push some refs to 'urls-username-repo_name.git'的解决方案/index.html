failed to push some refs to ‘urls:username/repo_name.git’的解决方案

标签（空格分隔）： Git

错误提示

解决方案

git pull –rebase

解决过程

提示里说更新被拒绝，因为你当前的分支在前面。大意就是你本地的仓库不是最新的，要和远程库合并之后再push,要你在使用git push orign master之前使用git pull合并远程仓库。
但是但我使用git pull之后，又出现了下面的错误。

要我去查看git-pull(1)这个网页,（原网页关闭了，找不到原图，）其中有一个图很关键，指出了原因。由于git pull是相当于是git fetch和git merge的，所以只需要使用git pull --rabase就可以解决了。
